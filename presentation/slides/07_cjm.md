# Слайд 7. Путь предпринимателя (CJM) по 3 подсегментам

**Одна мысль.** Путь новичка 17–25 одинаков по логике, но **разный по деталям в трёх подсегментах** — и на каждом этапе конструктор ставит свой модуль и продукт Платёжного бизнеса; кризис (этап 5) — точка, где решается «отток = закрытие».

**Носители (три платёжных сценария):** бьюти-мастер (SoftPOS) · селлер WB/Ozon (эквайринг) · репетитор/микрошкола (оплата по ссылке).

| Этап | Что происходит | Модуль коробки / продукт | Роль ИИ |
|---|---|---|---|
| −1 Предзапуск | идея есть, бизнеса и денег нет | AI-симуляция «собери бизнес», вход с подростковой карты | симуляция на данных эквайринга по локациям |
| 0–1 Легализация | выбор формы, регистрация | лестница «самозанятость с 14 → ИП», счёт из окна | подбор формы/режима по параметрам |
| 2 Инфраструктура | пик разрозненности (5–6 сервисов) | стек вертикали из коробки; ссылка = разметка для услуг | рекомендатор тарифа, автосверка касса↔учёт |
| 3 Первые продажи | первые чеки, тревога «дошли ли деньги» | авто-PnL из потока (прибыль, не остаток) | «AI-операционный директор» включается |
| 4 Рутина и рост | растёт операционка, нет кредитной истории | оборотка по скорингу потока, массовые выплаты | бенчмарки по похожим точкам, действия по команде |
| **5 Кризис** | падение спроса, кассовый разрыв | **подушка не-падения: резерв → овердрафт → аванс** | **сигнал по вертикали + проактивная помощь (opt-in)** |
| 6А Рост | второй мастер/точка/канал | зрелый клиент Платёжного бизнеса | — |
| 6Б Закрытие → возврат | ~90% готовы начать снова | контакт с человеком, память агента | вертикаль разворачивается заново за дни |

- **Разрозненность — сквозная боль этапов 0–4** у всех трёх; ответ — стек из коробки (тезис кейса про «единую логику»).
- **Этап 5 — самый недооценённый:** здесь клиента теряют, и здесь банк сегодня отсутствует проактивно. Механика разрыва **своя у каждого подсегмента** (сезонность / график выплат площадки / летний провал) — поэтому сигнал калибруется по вертикали, не один порог.
- **Потолок данных** у репетитора (личная карта → «для банка это просто входящие») снимается платёжной ссылкой как разметчиком — обязательный элемент вертикали.
