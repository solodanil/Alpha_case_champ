# Слайд 8. Что внутри коробки: ядро + вертикали

**Одна мысль.** Конструктор — это **одно платёжное ядро для всех** плюс **отраслевой пак под каждую вертикаль**; вертикаль открывается только если проходит фильтр из 6 критериев — поэтому модель масштабируется до десятков ниш, не распыляясь.

**Ядро (одинаково для всех вертикалей):** счёт → приём платежей (СБП / AlfaPay / эквайринг / Альфа-Касса) → авто-PnL из потока → подушка → AI-агент поверх всего.

**Три стартовых пака = три разных платёжных сценария** (по одному на выбранную отрасль):

| Вертикаль | Модули пака | Платёжный сценарий |
|---|---|---|
| Соло-бьюти-мастер / мини-студия | запись, расходники, чаевые | SoftPOS на телефоне, «Нет монет» |
| Селлер на WB / Ozon | закупка, юнит-экономика, оборотка | интернет-эквайринг своего канала + скоринг по обороту |
| Репетитор / микро-онлайн-школа | материалы, регулярные платежи, выплаты | оплата по ссылке-QR (AlfaPay, «Твои платежи»), Mozen |

- **Фильтр «подсегмент → вертикаль»** (6 порогов): массовость запусков среди 17–25, стандартизируемость (чек-лист покрывает ≥70–80% пути), зрелость вендоров, платёжная интенсивность, управляемость выживаемости, отсутствие лицензионных/возрастных барьеров. Показываем и **анти-примеры отсева**: автосервис падает на пороге входа, мобильная кофейня — на стандартизируемости локаций. Это доказывает, что «десятки вертикалей» — управляемый процесс, а не обещание «сделаем всё».
- **Глубина нарезки.** Вертикаль = общий стек + общий чек-лист; всё остальное — параметры, которые настраивает ИИ. Тест границы: если два бизнеса могут обменяться планом запуска, переделав <20%, — это одна вертикаль («студия у метро» и «в спальнике» — одна; «бьюти-студия» и «барбершоп» — разные).
- **Партнёрский слой, два уровня.** Якорные партнёры (1–2 на модуль): API-интеграция, SLA, revenue share, только они входят в «коробку». Вокруг — открытый маркетплейс с рейтингами, где банк даёт витрину, но не гарантирует.
- **Контроль качества — банковский.** Расчёты с партнёром идут через Альфу по модели **гарантированной сделки**: деньги поставщику после подтверждения поставки. Одновременно рычаг качества, защита новичка и дополнительный платёжный оборот.
- **Защита от копирования** — не эксклюзивные контракты (дорого, сужает выбор), а связка «транзакционные данные + бесшовная интеграция»; якорным достаточно маркетингового эксклюзива на 6–12 мес при запуске вертикали.
