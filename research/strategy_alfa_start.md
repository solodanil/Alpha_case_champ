# Стратегия «Альфа.Старт» — черновик (Алексей)

> **Статус:** предложение на обсуждение, не финал. Родилось из проработки идеи «CafeStore для других сфер» → развёрнуто в AI-продукт. Пересекается с веткой удержания команды (`branch_support_retention.md`) — это её передняя половина (привлечение), не альтернатива.

## Суть в одной строке

Банк привлекает молодых предпринимателей тем, что **помогает бизнесу родиться** — типовой AI-запуск по нише (шаблон + ИИ-проводник + приём платежей из коробки), без ручного труда специалистов. Дальше тот же ИИ ведёт бизнес и не даёт ему упасть. **Платёжные продукты — рельсы, ИИ — движок.**

## Как сюда пришли (эволюция идеи)

| Версия | Что было | Проблема | Фикс |
|--------|----------|----------|------|
| v1 | Франшиза под ключ: партнёры + лизинг техники ведут бизнес клиента | капиталоёмко, целится мимо 17–25, ИИ негде | — |
| v2 | То же, но как воронка привлечения («создаём бизнес → клиент наш») | CAC ≫ LTV: ручное создание дороже, чем платёжная выручка с молодого ИП | — |
| **v3 (текущая)** | **Типовой продукт, создаёт ИИ, специалисты руками не работают** | CAC ≈ 0, масштабируемо, ИИ = движок | ✅ встаёт по критериям |

Ключевой разворот: **партнёры с лизингом → типовой AI-продукт.** Именно он вытащил стратегию в рамки кейса.

## Схема

```
        17–25, есть идея, нет бизнеса, нет капитала, боится налогов
                                  │
        ┌─────────────────────────▼─────────────────────────┐
        │   1. СОЗДАТЬ  (привлечение)                        │
        │   «Альфа.Старт» по нише: бьюти / ритейл / услуги    │
        │   • ИИ-проводник: форма, налог, ОКВЭД, регистрация │
        │   • отраслевой шаблон (бизнес-план, бенчмарки)     │
        │   • приём платежей из коробки (СБП/QR, mPOS, ссылка)│
        │   → CAC ≈ 0, потому что делает ИИ, не человек       │
        └─────────────────────────┬─────────────────────────┘
                                  │  клиент банка с первого дня
        ┌─────────────────────────▼─────────────────────────┐
        │   2. РАБОТАТЬ  (ядро — платёжный бизнес + ИИ)      │
        │   AI-сопилот на данных эквайринга/кассы:           │
        │   • PnL и юнит-экономика, а не «остаток на счету»  │
        │   • напоминания о налогах, «отложи столько-то»     │
        │   • аналитика продаж по нише                       │
        └─────────────────────────┬─────────────────────────┘
                                  │
        ┌─────────────────────────▼─────────────────────────┐
        │   3. НЕ УПАСТЬ  (удержание — ветка команды)        │
        │   «Альфа видит и действует раньше»:               │
        │   • раннее предупреждение о кассовом разрыве/спаде │
        │   • подушка не-падения, отсрочка, «Подели»         │
        └─────────────────────────┬─────────────────────────┘
                                  │
     ЭФФЕКТ ДЛЯ БАНКА: дёшево привлёк → платёжная выручка с 1-го дня
     → удержал живым → LTV на годы → захват клиента на входе

     МОАТ: отраслевые шаблоны (партнёр оцифровал экспертизу 1 раз)
           + платёжные данные (чем больше, тем умнее ИИ)
```

## ✅ Сильные места

| # | Сила | Почему работает |
|---|------|-----------------|
| 1 | Захват на самом раннем этапе | Помогать *до* бизнеса — раньше «первого рубля». По кейсу: «полезен рано → сильная позиция» |
| 2 | ИИ — движок, а не довесок | Создание, ведение, предупреждение — всё на ИИ. Закрывает критерий №6 |
| 3 | Дешёвое масштабируемое привлечение | Типовой продукт + ИИ-проводник → CAC ≈ 0, «массовая история» реальна |
| 4 | Вертикальная специализация | Совпадает с ресёрчем (H9): нужен инструмент, понимающий нишу, а не общий суперапп |
| 5 | Сквозной путь под один нарратив | создать → работать → не упасть; смыкается с веткой удержания |
| 6 | Платёжные продукты — основа | СБП/эквайринг/касса — и данные для ИИ, и монетизация с первого дня |

## ⚠️ Слабые места и риски

| # | Риск | Крит. | Как закрывать |
|---|------|:---:|---------------|
| 1 | Моат: «ИИ объясняет» скопируют конкуренты | 🔴 | защита не в ИИ, а в **отраслевых шаблонах + платёжных данных**; партнёр оцифровывает нишу один раз |
| 2 | Обратный отбор — лёгкий старт притягивает самых слабых, а отток = закрытие | 🔴 | воронка окупается **только вместе с удержанием**; мёртвая база не платит |
| 3 | Юридическая ответственность ИИ за советы по налогам/регистрации | 🟡 | ИИ **информирует и передаёт в официальные потоки**, не даёт обязывающих советов |
| 4 | «Захватить все сферы» противоречит требованию фокуса | 🟡 | в кейсе — **2–3 ниши** как пилот; «массовость» = визия, не заявка |
| 5 | Не проверено, что уже есть у Альфы и у конкурентов | 🟡 | сверить с `competitors.md`, `analogi-banki-molodezh.md`, линейкой Платёжного бизнеса |
| 6 | ИИ-энтузиазм ЦА мог быть завышен нами | 🟢 | проверить на живых интервью (гипотеза H5, см. `bank_choice_criteria.md`) |

## Открытые вопросы (решить до сборки)

1. **ИИ проводит или делает за клиента?** (человек сам руками, ИИ подсказывает — vs ИИ оформляет, человек подтверждает). Определяет планку ответственности и «вау» для жюри.
2. **Роль партнёра-отраслевика** — только оцифровать нишу один раз, или есть и живая роль? Влияет на юнит-экономику.
3. **Какие 2–3 ниши** первыми под пилот — согласовать с `industry_selection.md`.
4. **Что из этого уже есть у Альфы** — чтобы не изобретать и не получить вопрос на защите.

## Связь с веткой удержания

Это **не альтернатива** нарративу «Альфа видит и действует раньше», а его **передняя половина**:

- **Привлечение (этот файл):** ИИ помогает бизнесу родиться → клиент на входе.
- **Удержание (`branch_support_retention.md`):** тот же ИИ потом не даёт упасть.

Один ИИ, один клиент, один путь: **создать → работать → не упасть.** Оба конца держит один AI-движок на платёжных данных.
